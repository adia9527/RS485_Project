# 基于 RS485 的多节点环境数据汇聚终端

## 1. 项目简介

本项目是一款面向智能家居场景的多节点环境数据汇聚终端，基于 STM32 主控实现多路 RS485 传感器数据采集、本地 LCD 图形化显示、参数配置、告警提示以及后续 Wi-Fi 数据上传扩展。

项目的核心目标是将分布在不同房间或区域的环境传感器数据进行统一采集、解析和展示，为智能家居系统提供更完整的环境感知能力。

与单个智能设备只采集自身附近数据不同，本项目更关注“多点位、多传感器、集中式管理”的场景。例如：

- 客厅、卧室、书房分别布置温湿度、CO₂、光照等传感器；
- 终端通过 RS485 总线轮询各节点数据；
- 本地屏幕实时显示环境状态；
- 当数据超过阈值时触发蜂鸣器、LED 或界面告警；
- 可通过 ESP32 将数据上传至上位机、服务器或智能家居平台。

## 10. RS485 通信设计

项目采用 RS485 总线连接外部传感器节点，通信协议以 Modbus RTU 为基础。

### Modbus RTU 请求帧示例

```text
设备地址 + 功能码 + 寄存器地址 + 寄存器数量 + CRC16
```

### 主要处理流程

1. 主机按照设备地址轮询传感器；
2. 发送 Modbus RTU 请求帧；
3. 等待传感器响应；
4. 校验 CRC16；
5. 解析寄存器数据；
6. 更新系统状态；
7. 若超时或 CRC 错误，则记录通信异常。

## 11. 参数保存设计

系统配置参数会保存到 Flash 中，支持掉电恢复。

主要保存内容包括：

- 设备 ID
- 温度上限 / 下限
- 湿度上限 / 下限
- CO₂ 阈值
- 光照阈值
- 告警开关
- 上传开关
- 屏幕背光设置

为了减少 Flash 擦写次数，项目采用 dirty flag 延迟保存机制：

```text
参数被修改
    │
    ▼
设置 dirty 标志
    │
    ▼
等待一段时间
    │
    ▼
若期间没有继续修改
    │
    ▼
写入 Flash
```

这种方式可以避免用户连续修改参数时频繁擦写 Flash。

## 12. LVGL 显示设计

本项目使用 LVGL 构建本地图形界面，界面主要包含：

- 实时环境数据卡片
- 传感器在线 / 离线状态
- 告警状态提示
- 参数配置页面
- 系统运行状态页面

LVGL 相关逻辑尽量与底层驱动解耦，显示层只负责读取应用层整理好的数据，不直接操作传感器或通信接口。

## 13. 项目特色

- 使用 STM32F429 + FreeRTOS + LVGL 构建完整嵌入式应用；
- 采用 RS485 总线，适合多节点、长距离、抗干扰数据采集；
- 使用 Modbus RTU 协议，贴近工业和智能设备通信场景；
- 软件采用 BSP / App / UI 分层架构，便于维护和扩展；
- 支持参数掉电保存，具备实际产品化思路；
- 预留 ESP32 Wi-Fi 上传接口，可扩展到物联网平台；
